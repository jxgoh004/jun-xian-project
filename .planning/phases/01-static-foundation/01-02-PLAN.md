  <action>
1. Ensure all docs/ files are committed and pushed to the main branch on GitHub:
   ```bash
   git add docs/
   git status
   git commit -m "feat: add static portfolio site structure for GitHub Pages"
   git push origin main
   ```

2. Enable GitHub Pages using the gh CLI, serving from the docs/ folder on the main branch:
   ```bash
   gh api repos/{owner}/{repo}/pages -X POST -f source.branch=main -f source.path=/docs
   ```
   If Pages is already enabled, update it:
   ```bash
   gh api repos/{owner}/{repo}/pages -X PUT -f source.branch=main -f source.path=/docs
   ```

3. Retrieve the GitHub Pages URL:
   ```bash
   gh api repos/{owner}/{repo}/pages --jq '.html_url'
   ```
   Record this URL -- it will be needed for the verification checkpoint.

4. Wait briefly and check the deployment status:
   ```bash
   gh api repos/{owner}/{repo}/pages/builds/latest --jq '.status'
   ```
   Expected: "built" (may take 1-2 minutes after enabling).

Note: If `gh` is not authenticated, this task becomes a checkpoint:human-action where the user enables Pages manually via GitHub Settings > Pages > Source: Deploy from branch > main > /docs. The executor should try the CLI first.
  </action>

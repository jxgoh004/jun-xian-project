## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Tag inline padding vertical (2px top/bottom = effectively 4px total) |
| sm | 8px | Card heading margin-bottom, tag gap |
| md | 16px | Default element spacing, tag horizontal padding |
| lg | 24px | Card internal padding, header horizontal padding, main horizontal padding |
| xl | 32px | — |
| 2xl | 48px | — |
| 3xl | 64px | Main vertical margin (60px, rounds to 64px token) |

Exceptions:
- FROZEN LEGACY — out-of-scope preservation only. Do not use for new elements. Header vertical padding is 14px — this value exists in the inherited `docs/index.html` CSS and MUST NOT be used for any new spacing decisions in Phase 2. Preserve as-is for out-of-scope legacy element only.
- FROZEN LEGACY — out-of-scope preservation only. Do not use for new elements. Card gap is 20px — this value exists in the inherited `docs/index.html` CSS and MUST NOT be used for any new spacing decisions in Phase 2. Preserve as-is for out-of-scope legacy element only.
- Tag border-radius is 12px (pill shape) — not a spacing value, preserve

All new spacing introduced in Phase 2 MUST use scale tokens only (multiples of 4). The frozen legacy values above are read-only carry-overs from existing code and are explicitly out of scope for reuse.

## Typography

| Role | Size | Weight | Line Height | Color token |
|------|------|--------|-------------|-------------|
| Body | 16px | 400 (regular) | 1.6 | --text (#c9d1d9) |
| Label / tag | 12px | 400 (regular) | 1.4 | --text-dim (#8b949e) |
| Card description | 14px | 400 (regular) | 1.5 | --text-dim (#8b949e) |
| Heading (card h3, header h1, section-label) | 16px | 600 (semibold) | 1.2 | --text-bright (#f0f6fc) |
| Display (hero name / page h2) | 28px | 600 (semibold) | 1.2 | --text-bright (#f0f6fc) |

Source: docs/index.html CSS — sizes and weights extracted directly. Card titles, header h1, and section-label achieve visual distinction from body text via font-weight 600 alone at 16px. Weights are 400 (regular) and 600 (semibold) only — no additional weights permitted in this phase.

## Color

All values extracted from existing `docs/index.html` `:root` CSS variables. Do not introduce new color values in this phase.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | #0d1117 (--bg) | Page background, tag background |
| Secondary (30%) | #161b22 (--surface) | Header background, card background |
| Border | #30363d (--border) | Card borders, header bottom border, tag borders |
| Accent (10%) | #58a6ff (--accent) | Card hover border-color only |
| Text primary | #c9d1d9 (--text) | Default body text |
| Text bright | #f0f6fc (--text-bright) | Headings, card titles, header site name |

Accent reserved for: card hover state border-color (`border-color: var(--accent)` on `.card:hover`) — nothing else. The LinkedIn contact link uses `--accent` as link color only (standard anchor styling). No other elements use the accent color.

## Component Inventory

Components that must exist in `docs/index.html` after Phase 2 completes:

### Hero Section
- Container: `<section class="hero">` inside `<main>`, above the `.projects` grid
- Layout: centered text (`text-align: center`)
- Elements (in order):
  1. Name — `<h2>` — 28px, weight 600, color `--text-bright`
  2. Title — `<p class="hero-title">` — 16px, weight 400, color `--accent`
  3. Bio paragraph — `<p class="hero-bio">` — 16px, weight 400, color `--text`, line-height 1.6, max-width 560px, centered
  4. Contact link — `<a href="https://linkedin.com/in/...">LinkedIn</a>` — 16px, color `--accent`, no underline by default, underline on hover
- Spacing: 48px bottom margin between hero and projects grid heading

### Project Card (extended from existing)
- Existing structure: `<a class="card" href="...">` — keep intact
- New: thumbnail `<img>` as first child inside `.card`, above `<h3>`, 16:9 aspect ratio, width 100%, border-radius 4px, margin-bottom 16px
- Placeholder (if thumbnail.png absent): `<div class="card-thumbnail-placeholder">` — CSS gradient block, same 16:9 dimensions (aspect-ratio: 16/9), background `linear-gradient(135deg, #1c2128, #30363d)`, border-radius 4px
- Existing card text content (h3, p, tags) unchanged

### Section heading
- `<h3 class="section-label">Projects</h3>` replacing existing `<h2>Projects</h2>` placeholder — 16px, weight 600, color `--text-bright`, text-align left, margin-bottom 16px
- Note: the current `<h2>Projects</h2>` will be repurposed; the display-size h2 slot is reserved for the hero name

## Accessibility Baseline

- All `<img>` elements must have descriptive `alt` attributes
- LinkedIn `<a>` must have visible link text (no icon-only links)
- Hero `<h2>` must be the first and only h2 on the page — establishes document outline
- Color contrast: `--text` (#c9d1d9) on `--bg` (#0d1117) passes WCAG AA (contrast ratio ~11.4:1)
- Color contrast: `--text-dim` (#8b949e) on `--surface` (#161b22) passes WCAG AA (contrast ratio ~4.6:1)
- Color contrast: `--accent` (#58a6ff) on `--bg` (#0d1117) passes WCAG AA for large text; acceptable for link text at 16px

# Milestone v2.0 -- Inside Bar Pattern Scanner -- Audit Report

## Executive Summary

Milestone v2.0 (Inside Bar Pattern Scanner) ships an end-to-end working product: the live deployed site at docs/projects/patterns/ reads a real data.json (44 detections, pipeline_status.completed: true, generated 2026-05-15 by three consecutive nightly cron runs), a real stats.json (D-13 fallback, n=1325), 41 annotated PNG charts on disk, and is wired into the portfolio home page card registry. All 11 cross-phase integration links checked are WIRED end-to-end on disk. All ROADMAP success criteria per phase are confirmed. **Verdict: tech_debt** -- the product works and all requirements are met, but two governance gaps require attention: (1) Phase 8 lacks an 08-VERIFICATION.md report (TRAIN-* requirements rest on plan-level summaries only), and (2) every phase VALIDATION.md still carries nyquist_compliant: false and wave_0_complete: false in frontmatter despite the phases shipping green -- these are stale draft markers that never got promoted on phase close. The REQUIREMENTS.md traceability table also shows DET/TRAIN/PIPE rows as Pending despite all upstream evidence saying complete (stale-marker concern explicitly flagged).

### Phase 7 (Detection Engine)
- **WR-01 (Warning):** _hh_hl_uptrend swing-pivot lookback scans full pre-confirmation slice rather than the documented trailing 60-bar window. More permissive than spec; filter still functions.
- **IN-01 / IN-02 / IN-04 / IN-05:** Cosmetic -- unused numpy import; mutable bars: List[Dict] inside frozen dataclass; 5-bar guarantee weak near right edge; CLI empty-args path untested.

### Phase 8 (Training Pipeline) -- primary governance gap
- **No 08-VERIFICATION.md exists.** All TRAIN-* coverage rests on plan-level SUMMARY frontmatter + downstream consumer evidence (Phase 9 imports split_config; Phase 10 loads ONNX successfully in production cron). Single most actionable remediation.
- **CONFIDENCE_FLOOR 0.5 -> 0.3 deviation** (user-approved). Phase 10 inference threshold must remain aligned; documented in 08-05-SUMMARY hand-off.
- **Cross-machine reproducibility caveat** -- manifest sha256 is per-machine equality, not cross-machine (matplotlib font rendering).
- **mAP@0.5:0.95 = 0.61** -- bbox tightness varies; future-milestone candidate for retrain with stronger box-loss weighting.

### Phase 9 (Backtesting Engine)
- **Empirical dev cache produced with --no-onnx** -- yolo_conf=null on all 40,022 records in _dev/backtest_cache.json. Per-record ONNX scoring in dev cache deferred; production nightly cron does populate yolo_conf via the live ONNX session.
- **Filter ablation surprise** -- unfiltered marginally outperforms filtered on out_of_sample; Phase 11 honest framing only (no UI toggle to expose the comparison).

### Phase 10 (Batch Pipeline)
- build_data_json 0 succeeded / 0 failed returns completed=False (docstring says vacuously True). No empty-universe test covers it.
- test_workflow_yaml.py has 4 tests vs plan-specified 5 (functionally equivalent).
- pipeline_status carries an additive schema_version: 1 not in the plan.

### Phase 11 (Frontend)
- No items flagged.

### Cross-cutting (Open questions from STATE.md still relevant)
- Spring (3-bar) vs full (5-bar) pattern: one or two class labels? -- current model treats as one class.
- Confidence-tier threshold revisit after ~30 nightly runs (D-01/D-02).
- numpy + onnxruntime compatibility -- currently locked to numpy < 2.0 transitively via ultralytics constraint.
